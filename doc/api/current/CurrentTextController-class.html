CurrentTextController

CurrentTextController<T> class

A TextEditingController that keeps a CurrentProperty and a text field in sync.

CurrentTextController is intended to remove the repetitive plumbing around reading, formatting, parsing, and writing text-field state when working with CurrentProperty values.

The controller supports:

  • String bindings to keep controller text in sync with the property's value
  • non-String bindings through a fromString parser
  • custom display formatting through asString
  • default values for cleared non-nullable fields
  • optional validation through CurrentFieldValidation
  • localized controller-generated validation issues through CurrentTextControllerValidationIssues

This controller does not replace your field validators. Instead, it fills in the gap between raw text editing and property-level validation by surfacing validation metadata for failures that occur before property-level validation, such as parse errors.
