CurrentTextControllerValidationIssues

CurrentTextControllerValidationIssues class final

Supplies controller-generated validation issues for CurrentTextController.

These issues are only used when a CurrentFieldValidation is attached to a controller and the controller itself needs to surface a validation problem that did not come from the property's validator rules.

Typical examples include generic messages such as "this value is required" or "this input could not be parsed".

Some concrete scenarios include:

  • the user entering text that fails parsing for a non-String property
  • the user clearing a non-nullable field that does not have a default value

This type exists so library consumers can localize those controller-level problems without forcing the controller to create localized strings inside the core package.

You can either:

  • override CurrentTextController.defaultValidationIssues once for the app
  • pass a per-binding validationIssues value to CurrentTextController.bind, CurrentTextController.bindString, CurrentTextController.bindInt, or CurrentTextController.bindDateTime

The builders are called the moment the validation problem occurs.

Example

ageController.bindInt(
  property: viewModel.age,
  lifecycleProvider: this,
  validationBuilder: (property, _) => property.createValidation(
    rules: [
      (value) => value < 18
          ? const CurrentValidationIssue(
              'profile.age.minimum',
              fallbackMessage: 'Must be at least 18',
            )
          : null,
    ],
    validateOnPropertyChange: true,
  ),
  validationIssues: CurrentTextControllerValidationIssues(
    requiredValueIssueBuilder: () => const CurrentValidationIssue(
      'profile.age.required',
      fallbackMessage: 'Entrez une valeur',
    ),
    invalidValueIssueBuilder: (_) => const CurrentValidationIssue(
      'profile.age.invalid',
      fallbackMessage: 'Valeur incorrecte',
    ),
  ),
);
